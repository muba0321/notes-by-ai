## 一、产品视角

### 1.1 需求背景

SRE 运维团队日常需要频繁访问各类运维平台（Prometheus、Grafana、Jenkins、Harbor 等），但：

#### 2.2.1 链接保存逻辑

```javascript
function saveLink() {
    // 1. 权限检查
    if (!authToken) { alert('请先登录'); return; }
    
    // 2. 表单验证
    var name = document.getElementById('form-name').value.trim();
    var url = document.getElementById('form-url').value.trim();
    if (!name || !url) { alert('请填写名称和地址'); return; }
    
    // 3. 构建数据
    var data = {
        icon: document.getElementById('form-icon').value || '🔗',
        name: name, url: url,
        description: document.getElementById('form-desc').value.trim(),
        category: document.getElementById('form-category').value,
        sort_order: parseInt(document.getElementById('form-sort').value) || 0
    };
    
    // 4. 发送请求
    var method = editingId ? 'PUT' : 'POST';
    var path = editingId ? '/api/v1/links/' + editingId : '/api/v1/links';
    fetch(path, {
        method: method,
        headers: {'Content-Type': 'application/json', 'Authorization': 'Bearer ' + authToken},
        body: JSON.stringify(data)
    }).then(function(r) { return r.json(); })
      .then(function(json) {
          if (json.code === 0) { 
              closeFormDialog(); 
              loadManageTable(); 
              loadQuickLinks(); 
          } else { 
              alert('保存失败: ' + (json.message || json.msg || '请求失败')); 
          }
      }).catch(function(err) { 
          alert('保存请求失败: ' + (err.message || '网络错误')); 
      });
}
```

### 2.3 踩坑记录

| 问题 | 现象 | 根因 | 解决方案 |
|------|------|------|---------|
| 保存按钮无响应 | 点击"确定"后弹窗不关闭 | 未检查 `authToken` 是否为空 | 添加 `if (!authToken)` 前置检查 |
| 错误提示为空 | `保存失败: undefined` | 后端返回 `{msg: "..."}` 但前端读 `json.message` | 改为 `json.message \|\| json.msg \|\| '请求失败'` |
| URL 列撑破表格 | 长链接导致表格溢出 | 未设置 `table-layout: fixed` 和 `max-width` | 添加固定布局 + 省略号截断 |
| 中文乱码 | 描述显示 `????` 或乱码字符 | 连接字符串未指定 `charset=utf8mb4` | 数据库连接加 `?charset=utf8mb4` |

### 3.2 数据库设计

```sql
-- 常用链接表
CREATE TABLE common_links (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(64) NOT NULL COMMENT '链接名称',
    url VARCHAR(512) NOT NULL COMMENT '目标地址',
    icon VARCHAR(32) DEFAULT '' COMMENT 'Emoji 图标',
    description VARCHAR(256) DEFAULT '' COMMENT '描述文案',
    category VARCHAR(32) DEFAULT 'default' COMMENT '分类标识',
    sort_order INT DEFAULT 0 COMMENT '排序权重',
    is_enabled TINYINT(1) DEFAULT 1 COMMENT '是否启用',
    is_visible TINYINT(1) DEFAULT 1 COMMENT '是否前端可见',
    roles VARCHAR(256) DEFAULT '' COMMENT '可见角色（逗号分隔）',
    created_at DATETIME,
    updated_at DATETIME,
    created_by VARCHAR(64) DEFAULT '',
    updated_by VARCHAR(64) DEFAULT '',
    INDEX idx_category (category),
    INDEX idx_sort (sort_order, is_enabled),
    INDEX idx_visible (is_visible, is_enabled)
) DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;

-- 变更日志表
CREATE TABLE link_change_log (
    id INT AUTO_INCREMENT PRIMARY KEY,
    link_id INT NOT NULL COMMENT '关联链接 ID',
    action VARCHAR(16) NOT NULL COMMENT '操作类型 create/update/delete',
    old_data JSON COMMENT '变更前数据',
    new_data JSON COMMENT '变更后数据',
    operator VARCHAR(64) NOT NULL COMMENT '操作人',
    created_at DATETIME,
    INDEX idx_log_link (link_id),
    INDEX idx_log_created (created_at)
) DEFAULT CHARSET=utf8mb4;
```

### 3.4 关键踩坑

#### 问题 1：JWT Token 创建失败

**现象：** 新增链接时返回 `{"msg": "Subject must be a string"}`

**根因：** Flask-JWT-Extended 4.x 要求 `identity` 必须是字符串，但代码传入字典：
```python
# ❌ 错误写法
identity = {"sub": username, "roles": ["admin"]}
token = create_access_token(identity=identity)
```

**修复：**
```python
# ✅ 正确写法
token = create_access_token(identity=username, additional_claims={"roles": ["admin"]})
```

#### 问题 2：MySQL 中文乱码

**现象：** 描述字段显示 `????` 或乱码字符

**根因：** 数据库连接未指定字符集，默认使用 latin1

**修复：**
```python
# docker-compose.yml
DATABASE_URL: mysql+pymysql://root:sre2026@mysql:3306/sre_portal?charset=utf8mb4

# app.py
app.config["SQLALCHEMY_DATABASE_URI"] = "mysql://root:sre2026@mysql:3306/sre_portal?charset=utf8mb4"
```

#### 问题 3：Flask-JWT-Extended 响应格式不统一

**现象：** 权限校验失败返回 `{"msg": "Missing Authorization Header"}`，但成功响应使用 `{"code": 0, "message": "..."}`

**根因：** JWT 扩展使用 `msg` 字段，业务代码使用 `message` 字段

**修复：** 前端错误处理兼容两种格式：
```javascript
json.message || json.msg || '请求失败'
```
