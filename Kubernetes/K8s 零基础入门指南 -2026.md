### 市场现状（2025-2026）

| 指标 | 数据 |
|------|------|
| **岗位需求** | DevOps/云原生岗位 90% 要求 K8s 技能 |
| **平均薪资** | K8s 工程师 ¥35K-60K/月 |
| **企业采用率** | 全球 500 强 83% 已采用 K8s |
| **云原生趋势** | 容器化部署成为标准 |

### K8s 能解决什么问题？

**传统部署的问题：**
```
┌─────────────────────────────────────────────────────────┐
│  问题 1：应用部署复杂                                    │
│  - 手动配置环境、依赖、网络                              │
│  - 容易出错，难以复现                                    │
├─────────────────────────────────────────────────────────┤
│  问题 2：扩缩容困难                                      │
│  - 流量高峰时手动加机器                                  │
│  - 流量低谷时资源浪费                                    │
├─────────────────────────────────────────────────────────┤
│  问题 3：故障恢复慢                                      │
│  - 服务器宕机后手动迁移应用                              │
│  - 恢复时间长，影响业务                                  │
├─────────────────────────────────────────────────────────┤
│  问题 4：资源利用率低                                    │
│  - 每台服务器独立部署                                    │
│  - 资源无法共享，浪费严重                                │
└─────────────────────────────────────────────────────────┘
```

**K8s 的解决方案：**
```
┌─────────────────────────────────────────────────────────┐
│  ✅ 自动化部署     - YAML 定义，一键部署                  │
│  ✅ 自动扩缩容     - 根据 CPU/内存自动增减 Pod            │
│  ✅ 自愈能力       - Pod 宕机自动重启/重新调度            │
│  ✅ 资源优化       - 多应用共享节点，提高利用率           │
│  ✅ 服务发现       - 自动分配 DNS 和负载均衡               │
│  ✅ 滚动更新       - 零停机更新应用版本                   │
└─────────────────────────────────────────────────────────┘
```

### Docker 快速回顾

```bash
# 如果你还没学过 Docker，先完成这些练习：

# 1. 理解镜像和容器
docker pull nginx              # 下载镜像
docker run -d nginx            # 运行容器
docker ps                      # 查看运行中的容器

# 2. 理解端口映射

# 4. 查看日志
```

### 10 个核心概念（必须理解）

| 概念 | 比喻 | 说明 |
|------|------|------|
| **Cluster（集群）** | 整个公司 | 一组 Node 的集合，共同工作 |
| **Node（节点）** | 员工工位 | 运行 Pod 的服务器（物理机/虚拟机） |
| **Pod** | 办公室（可多人） | K8s 最小调度单元，可包含 1 个或多个容器 |
| **Deployment** | 项目经理 | 确保指定数量的 Pod 始终运行 |
| **Service** | 前台/总机 | 稳定的网络入口，负载均衡 |
| **Namespace** | 部门 | 逻辑隔离，不同团队用不同 Namespace |
| **ConfigMap** | 配置文件 | 存储非敏感配置（如环境变量） |
| **Secret** | 保险柜 | 存储敏感信息（密码、密钥） |
| **Volume** | 共享硬盘 | 持久化存储，Pod 重启数据不丢失 |
| **Ingress** | 公司大门 | 外部访问集群的入口（HTTP/HTTPS） |

### 方案 1：Minikube（推荐新手）⭐

**特点：** 单机运行 K8s，最简单

#### 安装步骤（Linux/Ubuntu）

```bash
# 1. 安装 Docker（如果还没装）
curl -fsSL https://get.docker.com | sh

# 2. 安装 Minikube
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
sudo install minikube-linux-amd64 /usr/local/bin/minikube

# 3. 安装 kubectl（K8s 命令行工具）
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
sudo install kubectl /usr/local/bin/kubectl

# 4. 启动 Minikube
minikube start --driver=docker --cpus=2 --memory=4096

# 5. 验证安装
kubectl get nodes
# 输出：minikube   Ready   control-plane   <version>

# 6. 启用 Dashboard（可视化界面）
minikube dashboard
```

#### 常用命令

```bash
minikube start          # 启动集群
minikube stop           # 停止集群
minikube delete         # 删除集群
minikube status         # 查看状态
minikube dashboard      # 打开可视化界面
```

### 方案 2：Kind（Kubernetes in Docker）⭐⭐

**特点：** 用 Docker 容器模拟 K8s 集群，适合测试多节点

```bash
# 1. 安装 Kind
curl -Lo ./kind https://kind.sigs.k8s.io/dl/v0.20.0/kind-linux-amd64
chmod +x ./kind
sudo mv ./kind /usr/local/bin/kind

# 2. 创建集群（默认 1 个控制节点 + 1 个工作节点）
kind create cluster --name my-cluster

# 3. 创建多节点集群（1 个 master + 3 个 worker）
cat <<EOF | kind create cluster --config=-
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
nodes:
- role: control-plane
- role: worker
- role: worker
- role: worker
EOF

# 4. 查看集群
kubectl get nodes

# 5. 删除集群
kind delete cluster --name my-cluster
```

### 方案 3：kubeadm（生产环境）⭐⭐⭐⭐

**特点：** 官方推荐的生产环境部署工具

#### 环境要求

| 角色 | 数量 | 配置 | 系统 |
|------|------|------|------|
| Master | 1 台（或 3 台高可用） | 2 核 4G | Ubuntu 20.04+ |
| Worker | 至少 1 台（推荐 2+） | 2 核 2G | Ubuntu 20.04+ |

#### 快速部署脚本

```bash
# ⚠️ 所有节点执行（Master + Worker）

# 1. 关闭防火墙/swap
systemctl stop firewalld && systemctl disable firewalld
swapoff -a
sed -i '/swap/s/^/#/' /etc/fstab

# 2. 配置内核参数
cat > /etc/sysctl.d/k8s.conf <<EOF
net.bridge.bridge-nf-call-ip6tables = 1
net.bridge.bridge-nf-call-iptables = 1
net.ipv4.ip_forward = 1
EOF
sysctl --system

# 3. 安装 containerd
apt update && apt install -y containerd
mkdir -p /etc/containerd
containerd config default > /etc/containerd/config.toml
sed -i 's/SystemdCgroup = false/SystemdCgroup = true/' /etc/containerd/config.toml
systemctl restart containerd

# 4. 安装 K8s 组件（使用阿里云镜像）
curl https://mirrors.aliyun.com/kubernetes/apt/doc/apt-key.gpg | apt-key add -
cat <<EOF > /etc/apt/sources.list.d/kubernetes.list
deb https://mirrors.aliyun.com/kubernetes/apt/ kubernetes-xenial main
EOF
apt update
apt install -y kubelet kubeadm kubectl

# ===== 仅 Master 节点执行 =====
# 初始化集群
kubeadm init --image-repository=registry.aliyuncs.com/google_containers \
  --pod-network-cidr=10.244.0.0/16

# 配置 kubectl
mkdir -p $HOME/.kube
cp -i /etc/kubernetes/admin.conf $HOME/.kube/config

# 安装网络插件（Calico）
kubectl apply -f https://docs.projectcalico.org/manifests/calico.yaml

# ===== 仅 Worker 节点执行 =====
# 加入集群（命令从 Master 节点获取）
kubeadm join <MASTER-IP>:6443 --token <TOKEN> --discovery-token-ca-cert-hash sha256:<HASH>
```

## 📦 核心资源对象详解

### 1. Pod（最小调度单元）

```yaml
# pod-example.yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
  labels:
    app: web
spec:
  containers:
  - name: nginx
    image: nginx:1.21
    ports:
    - containerPort: 80
    resources:
      requests:
        memory: "64Mi"
        cpu: "250m"
      limits:
        memory: "128Mi"
        cpu: "500m"
```

**部署命令：**
```bash
kubectl apply -f pod-example.yaml
kubectl get pods
kubectl delete pod nginx-pod
```

---

### 2. Deployment（无状态应用）

```yaml
# deployment-example.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3                    # 期望的 Pod 数量
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.21
        ports:
        - containerPort: 80
```

**常用操作：**
```bash
# 创建
kubectl apply -f deployment-example.yaml

# 查看状态
kubectl get deployments
kubectl get pods

# 扩缩容
kubectl scale deployment nginx-deployment --replicas=5

# 更新镜像
kubectl set image deployment/nginx-deployment nginx=nginx:1.22

# 回滚
kubectl rollout undo deployment/nginx-deployment

# 查看滚动更新状态
kubectl rollout status deployment/nginx-deployment
```

---

### 3. Service（服务发现/负载均衡）

```yaml
# service-example.yaml
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx                    # 选择带有 app=nginx 标签的 Pod
  ports:
  - port: 80                      # Service 暴露的端口
    targetPort: 80                # Pod 内容器的端口
  type: ClusterIP                 # 类型：ClusterIP/NodePort/LoadBalancer
```

**Service 类型对比：**

| 类型 | 说明 | 访问范围 | 使用场景 |
|------|------|---------|---------|
| **ClusterIP** | 集群内 IP | 集群内部 | 内部服务通信 |
| **NodePort** | 节点端口 | 通过节点 IP 访问 | 测试/开发环境 |
| **LoadBalancer** | 云负载均衡 | 公网访问 | 生产环境对外服务 |
| **ExternalName** | 外部服务 | 集群内外 | 访问外部数据库等 |

---

### 4. ConfigMap（配置管理）

```yaml
# configmap-example.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  APP_ENV: production
  LOG_LEVEL: info
  database.conf: |
    host=db.example.com
    port=5432
    name=myapp
```

**使用 ConfigMap：**
```yaml
# 在 Deployment 中使用
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  template:
    spec:
      containers:
      - name: app
        image: my-app:1.0
        envFrom:
        - configMapRef:
            name: app-config
        volumeMounts:
        - name: config-volume
          mountPath: /etc/config
      volumes:
      - name: config-volume
        configMap:
          name: app-config
```

### 任务 3：服务暴露与访问 ⭐⭐

**目标：** 理解 Service

```bash
# 1. 创建 Deployment
kubectl create deployment nginx --image=nginx:1.21

# 2. 暴露为 Service（NodePort 类型）
kubectl expose deployment nginx --type=NodePort --port=80

# 3. 查看 Service
kubectl get services

# 4. 获取访问地址
minikube service nginx --url

# 5. 浏览器访问或使用 curl
curl <URL>
```

**✅ 完成标准：** 浏览器看到 Nginx 欢迎页面

### 任务 4：滚动更新 ⭐⭐

**目标：** 理解零停机更新

```bash
# 1. 创建 Deployment（版本 1.21）
kubectl create deployment nginx --image=nginx:1.21 --replicas=3

# 2. 查看当前镜像
kubectl get deployment nginx -o jsonpath='{.spec.template.spec.containers[0].image}'

# 3. 更新到版本 1.22
kubectl set image deployment/nginx nginx=nginx:1.22

# 4. 观察滚动更新过程
kubectl rollout status deployment/nginx
kubectl get pods -w

# 5. 回滚到版本 1.21
kubectl rollout undo deployment/nginx

# 6. 查看更新历史
kubectl rollout history deployment/nginx
```

**✅ 完成标准：** 完成更新和回滚，服务始终可用

### 任务 7：完整项目 ⭐⭐⭐

**目标：** 综合运用所有知识

**需求：** 部署一个带配置管理的 Web 应用

```yaml
# complete-app.yaml
---
apiVersion: v1
kind: ConfigMap
metadata:
  name: web-config
data:
  WELCOME_MSG: "Hello from Kubernetes!"
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: web
  template:
    metadata:
      labels:
        app: web
    spec:
      containers:
      - name: nginx
        image: nginx:1.21
        ports:
        - containerPort: 80
        envFrom:
        - configMapRef:
            name: web-config
---
apiVersion: v1
kind: Service
metadata:
  name: web-service
spec:
  selector:
    app: web
  ports:
  - port: 80
    targetPort: 80
  type: NodePort
```

**部署命令：**
```bash
kubectl apply -f complete-app.yaml
kubectl get all
minikube service web-service --url
```

**✅ 完成标准：** 成功部署并访问应用

## 🐛 常见问题排查

### 问题 1：Pod 一直处于 Pending 状态

**可能原因：**
- 资源不足（CPU/内存）
- 没有可用的 Node
- PVC 未绑定

**排查步骤：**
```bash
kubectl describe pod <pod-name>
kubectl get nodes
kubectl top nodes
```

---

### 问题 2：Pod 状态为 CrashLoopBackOff

**可能原因：**
- 应用启动失败
- 配置错误
- 缺少依赖

**排查步骤：**
```bash
kubectl describe pod <pod-name>
kubectl logs <pod-name>
kubectl logs <pod-name> --previous
```

---

### 问题 3：ImagePullBackOff

**可能原因：**
- 镜像名称错误
- 镜像不存在
- 网络问题

**排查步骤：**
```bash
kubectl describe pod <pod-name>
# 检查镜像名称是否正确
# 尝试手动拉取镜像
docker pull <image-name>
```

---

### 问题 4：Service 无法访问

**可能原因：**
- Selector 不匹配
- 端口配置错误
- 网络插件问题

**排查步骤：**
```bash
kubectl get service <service-name>
kubectl get endpoints <service-name>
kubectl get pods --show-labels
# 检查 Pod 标签是否与 Service 的 selector 匹配
```

---

### 问题 5：kubectl 命令报错

**常见错误：**
```bash
# 错误：The connection to the server localhost:8080 was refused
# 解决：检查 kubeconfig
export KUBECONFIG=$HOME/.kube/config

# 错误：Unable to connect to the server: dial tcp: lookup ...
# 解决：检查网络/DNS
ping <apiserver-ip>
```
