## 🏗️ 架构设计

### Agent 角色

| Agent | 职责 | 飞书应用 | 位置 |
|-------|------|---------|------|
| **main** | 总协调 | openclaw-总助手 | 主节点 |
| **consult** | 业务咨询 | openclaw-咨询 | 主节点 |
| **coding** | 代码生成 | openclaw-开发 | 子节点 1 |
| **product** | 产品设计 | (新建) | 子节点 1 |
| **monitor** | 监控告警 | (新建) | 堡垒机 |

### 协作模式

```
链式协作：用户 → main → consult → main → 用户
并行协作：用户 → main → [consult + coding + monitor] → main → 用户
接力协作：用户 → main → coding → product → monitor → 用户
```

---

## 📊 沟通留痕三层架构

```
┌──────────────────┐     ┌──────────────────┐     ┌──────────────────┐
│  第一层：飞书消息  │     │  第二层：多维表格  │     │  第三层：Git 归档  │
│  (实时对话)       │────▶│  (任务/状态)      │────▶│  (文档/报告)      │
└──────────────────┘     └──────────────────┘     └──────────────────┘
```

### 飞书多维表格（3 个核心表）

1. **任务队列表** - 任务追踪、状态管理
2. **Agent 状态表** - 在线/离线、负载监控
3. **沟通记录表** - Agent 间对话存档
