## 🎯 为什么学习云平台

### 市场需求（2025-2026）

| 岗位 | 平均薪资（国内） | 增长率 | 远程工作 |
|------|-----------------|--------|---------|
| 云工程师 | ¥25K-45K/月 | +45% | 95% |
| DevOps 工程师 | ¥30K-50K/月 | +42% | 90% |
| SRE 工程师 | ¥35K-60K/月 | +50% | 85% |
| 云架构师 | ¥50K-80K/月 | +38% | 80% |

**数据来源：** 2025 全球云服务市场报告

### 云平台的核心价值

1. **弹性伸缩** - 按需使用，不用前期重资产投入
2. **全球部署** - 分钟级开通全球节点
3. **高可用** - 内置冗余、备份、容灾能力
4. **自动化** - API 驱动，一切皆可代码化
5. **成本优化** - 按量付费 + 自动缩容

### 第 2 个月：云计算核心概念

#### 核心概念

| 概念 | 说明 | 类比 |
|------|------|------|
| **IaaS** | 基础设施即服务 | 租毛坯房，自己装修 |
| **PaaS** | 平台即服务 | 租精装房，直接入住 |
| **SaaS** | 软件即服务 | 住酒店，拎包入住 |
| **Region** | 地域 | 城市（如北京、上海） |
| **AZ** | 可用区 | 城市里的不同机房 |
| **VPC** | 私有网络 | 你家的围墙和院子 |
| **Subnet** | 子网 | 房间 |
| **Security Group** | 安全组 | 门锁和门禁 |

```bash
# 4. 学习计费方式：
- 按量付费 vs 包年包月
```

### 第 5 个月：CI/CD 与自动化

#### CI/CD 核心概念

| 概念 | 说明 | 工具 |
|------|------|------|
| **CI** | 持续集成 | Jenkins/GitHub Actions |
| **CD** | 持续交付 | ArgoCD/Spinnaker |
| **GitOps** | Git 驱动的运维 | ArgoCD/Flux |
| **IaC** | 基础设施即代码 | Terraform/Pulumi |

```
代码提交 → 代码审查 → 自动化测试 → 构建镜像 → 
安全扫描 → 部署测试环境 → 自动化测试 → 
部署生产环境 → 监控告警
```

```bash
- 编写 workflow 文件

- 集成云厂商 CLI

# 3. 学习 Terraform：
- 编写 .tf 文件
- 创建 VPC/ECS/RDS
- 状态文件管理
```

### 第 6 个月：监控与 SRE

#### 监控体系

| 层级 | 指标 | 工具 |
|------|------|------|
| **基础设施** | CPU/内存/磁盘/网络 | Prometheus/云监控 |
| **应用层** | QPS/延迟/错误率 | SkyWalking/Pinpoint |
| **业务层** | 订单量/转化率 | 自定义埋点 |
| **日志** | 访问日志/错误日志 | ELK/Loki |
| **链路追踪** | 请求链路分析 | Jaeger/Zipkin |

#### SRE 核心概念

| 概念 | 说明 | 公式 |
|------|------|------|
| **SLI** | 服务等级指标 | 衡量指标（如可用性 99.9%） |
| **SLO** | 服务等级目标 | 目标值（如 99.95%） |
| **SLA** | 服务等级协议 | 合同承诺（如 99.9%，赔偿条款） |
| **错误预算** | 允许的故障时间 | (100% - SLO) × 时间周期 |

#### 本周任务

```bash
# 1. 搭建 Prometheus + Grafana：
- 部署 Prometheus Server
- 配置 Node Exporter
- 导入 Grafana Dashboard
- 设置告警规则

# 2. 学习 SRE 实践：
- 定义 SLI/SLO
- 计算错误预算
- 制定应急预案

# 3. 学习混沌工程：
- Chaos Mesh / ChaosBlade
- 注入故障（网络延迟/Pod 删除）
- 验证系统韧性
```

- 📖 《SRE：Google 运维解密》
- 📖 《SRE 实战手册》
- 🌐 https://sre.google/

## 🎯 学习建议

### ✅ 应该做的

1. **动手实践** - 云是实践出来的，不是看出来的
2. **记录笔记** - 用飞书/Notion 记录学习过程
3. **参与社区** - GitHub/知乎/Stack Overflow
4. **考取认证** - 系统化学习 + 简历加分
5. **关注成本** - 设置预算告警，避免意外费用

### ❌ 不应该做的

1. **不要只看不练** - 理论 + 实践 = 真本事
2. **不要盲目追求新技术** - 基础最重要
3. **不要忽视安全** - 权限/密钥/网络隔离
4. **不要忽略成本** - 云资源很贵，用完及时释放
5. **不要单打独斗** - 加入社区，多交流
