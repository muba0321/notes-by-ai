## 1. 变更概述

### 1.1 变更背景

原有 Grafana 监控存在以下问题：
1. 仅监控堡垒机单一节点
2. Dashboard 指标数量不足（<10 个）
3. 无节点选择器，无法切换查看不同节点
4. 主节点和子节点 1 未纳入监控

### 1.2 变更目标

1. 扩容监控指标至 16+ 个面板
2. 实现三节点统一监控（堡垒机、主节点、子节点 1）
3. 配置节点选择器，支持切换查看
4. 建立双监控中心架构，提高可用性

### 1.3 变更范围

| 系统 | 变更内容 | 影响 |
|------|----------|------|
| Prometheus | 配置多节点 targets | 无停机 |
| Grafana | Dashboard 扩容 + 数据源配置 | 无停机 |
| Node Exporter | 主节点/子节点 1 部署 | 无停机 |
| 网络配置 | 使用外网 IP 替代内网 IP | 无影响 |

## 5. 遇到的问题与解决方案

### 5.1 问题 1：Grafana 页面加载失败

**现象：** 页面显示"Grafana has failed to load its application files"

**原因：** `serve_from_sub_path` 配置未启用

**解决方案：**
```yaml
environment:
  - GF_SERVER_SERVE_FROM_SUB_PATH=true
```

**结果：** ✅ 页面正常加载

---

### 5.2 问题 2：主节点内网 IP 不可达

**现象：** Prometheus target 状态 down，timeout 无法连接

**原因：** 堡垒机与主节点之间内网 IP (10.0.118.4) 无法跨网段访问

**排查过程：**
```bash
# 堡垒机测试
timeout 3 bash -c 'echo > /dev/tcp/10.0.118.4/9100'  # 失败

# 子节点 1 测试
timeout 3 bash -c 'echo > /dev/tcp/10.0.118.4/9100'  # 成功

# 主节点外网 IP 测试
curl -s http://38.246.245.32:9100/metrics  # 成功
```

**解决方案：** 使用外网 IP (38.246.245.32) 替代内网 IP

**结果：** ✅ 所有节点可达

---

### 5.3 问题 3：Grafana 容器权限错误

**现象：** 容器重启，日志显示"Permission denied"

**原因：** 挂载目录权限不足

**解决方案：**
```bash
chmod -R 777 /data/monitoring/grafana/data
docker restart grafana
```

**结果：** ✅ 容器正常运行

---

### 5.4 问题 4：节点选择器无选项

**现象：** Dashboard 顶部"节点"下拉菜单为空

**原因：** PromQL 查询 `label_values(node_uname_info, instance)` 无结果

**排查：**
```bash
curl -s 'http://localhost:9090/api/v1/query?query=node_uname_info'
# 返回空结果
```

**根本原因：** Prometheus 数据源未正确配置或 targets 未采集到指标

**解决方案：**
1. 确认 Prometheus 数据源 URL 正确
2. 验证 targets 状态全部 up
3. 等待指标采集（约 30s）

**结果：** ✅ 节点选择器显示 3 个选项
