## 🚀 部署步骤

### 步骤 1：所有节点执行预备脚本

```bash
# 在 master1、node1、node2 上分别执行
sshpass -p 'Huanxin0321' ssh root@124.132.136.17 -p [端口] << 'EOF'
# 系统配置
cat > /etc/sysctl.d/k8s.conf << SYSEOF
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
SYSEOF
sysctl --system

# 安装 containerd
apt-get update
apt-get install -y containerd
mkdir -p /etc/containerd
containerd config default > /etc/containerd/config.toml
sed -i 's/SystemdCgroup = false/SystemdCgroup = true/' /etc/containerd/config.toml
systemctl restart containerd

# 安装 Kubernetes
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.29/deb/Release.key | apt-key add -
echo 'deb https://pkgs.k8s.io/core:/stable:/v1.29/deb/ /' > /etc/apt/sources.list.d/kubernetes.list
apt-get update
apt-get install -y kubelet kubeadm kubectl
apt-mark hold kubelet kubeadm kubectl
systemctl enable kubelet
EOF
```

### 步骤 2：Master 节点初始化

```bash
# 在 master1 执行
kubeadm init \
  --pod-network-cidr=10.244.0.0/16 \
  --apiserver-advertise-address=0.0.0.0 \
  --control-plane-endpoint=124.132.136.17:9005

# 配置 kubectl
mkdir -p $HOME/.kube
cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
chown $(id -u):$(id -g) $HOME/.kube/config

# 安装 Calico CNI
kubectl apply -f https://raw.githubusercontent.com/projectcalico/calico/v3.27.0/manifests/calico.yaml
```

### 步骤 3：Worker 节点加入

```bash
# 在 node1 和 node2 执行
# 使用 master1 初始化后输出的 kubeadm join 命令
kubeadm join 124.132.136.17:9005 \
  --token [token] \
  --discovery-token-ca-cert-hash sha256:[hash]
```

### 步骤 4：验证集群

```bash
# 在 master1 执行
kubectl get nodes
kubectl get pods --all-namespaces
```

## ⚠️ 当前问题

**SSH 连接超时**

可能原因：
1. SSH 服务未启动
2. 防火墙阻止 SSH
3. 网络延迟过高

解决步骤：
```bash
# 1. 检查 SSH 服务状态
sshpass -p 'Huanxin0321' ssh root@124.132.136.17 -p 9005 "systemctl status ssh"

# 2. 检查防火墙
sshpass -p 'Huanxin0321' ssh root@124.132.136.17 -p 9005 "ufw status"

# 3. 尝试增加超时时间
sshpass -p 'Huanxin0321' ssh -o ConnectTimeout=60 root@124.132.136.17 -p 9005 "hostname"
```
