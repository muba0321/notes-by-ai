## 🔍 根本原因

### 问题 1: 端口映射导致 TLS 失败

**现象:**
```
tls: first record does not look like a TLS handshake
```

**原因:**
- K8s API Server 默认使用 HTTPS (6443 端口)
- 当前通过端口 9005 映射，但该端口配置为 HTTP
- kubelet 尝试 HTTPS 连接但收到 HTTP 响应

**解决方案:**
1. 修改端口映射配置，将 6443 映射到 9005
2. 或在路由器配置 NAT 规则
3. 或使用内网 IP 直接访问

### 问题 2: kubeadm init 无法选择 IP

**现象:**
```
unable to select an IP from lo network interface
```

**原因:**
- 系统只有 lo (localhost) 网络接口
- 没有配置实际的以太网接口 IP

**解决方案:**
1. 配置虚拟网络接口
2. 或指定 `--apiserver-advertise-address=127.0.0.1`
3. 或使用内网 IP

## 🛠️ 建议方案

### 方案 A: 使用内网 IP 部署 (推荐)

如果 K8s 节点有内网 IP：

```bash
# 在 master1 执行
kubeadm init \
  --pod-network-cidr=10.244.0.0/16 \
  --apiserver-advertise-address=<内网 IP> \
  --image-repository=registry.aliyuncs.com/google_containers \
  --kubernetes-version=v1.28.2
```

### 方案 B: 配置端口映射

在路由器配置：
```
外部 9005 → 内部 124.132.136.17:6443 (TCP)
外部 9191 → 内部 124.132.136.17:6443 (TCP)
外部 9053 → 内部 124.132.136.17:6443 (TCP)
```

### 方案 C: 使用 k3s (最简单)

k3s 单二进制，无需复杂网络配置：

```bash
# master1
curl -sfL https://get.k3s.io | sh -s - server --node-ip 124.132.136.17 --https-listen-port 9005

# node1/node2
curl -sfL https://get.k3s.io | sh -s - agent --server https://124.132.136.17:9005
```
