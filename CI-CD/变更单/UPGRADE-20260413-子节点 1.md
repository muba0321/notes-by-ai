## 二、升级逻辑

### 升级方式

OpenClaw 使用 npm 包管理，升级逻辑：

```bash
# 1. 停止 Gateway 服务
openclaw gateway stop

# 2. 全局更新 npm 包
npm install -g openclaw@latest

# 3. 验证版本
openclaw --version

# 4. 重启 Gateway
openclaw gateway start

# 5. 检查状态
openclaw gateway status
```

### 配置保留

| 配置类型 | 位置 | 是否保留 |

## 七、问题与解决

### 问题 1: RPC Probe 超时

**现象：**
```
RPC probe: failed
RPC target: ws://127.0.0.1:18789
  gateway closed (1000): 
```

**原因：** Gateway 重启后 WebSocket 需要几秒初始化时间

**解决：** 等待 5 秒后重试，Gateway 正常运行

**状态：** ✅ 已解决（非实质性问题）

### 问题 2: journalctl 无日志

**现象：**
```
No journal files were found.
```

**原因：** 子节点 1 systemd user 日志未配置持久化

**影响：** 不影响运行，日志存储在 `/tmp/openclaw/openclaw-2026-04-13.log`

**状态：** ⚠️ 已知限制（可接受）

### 成功经验

1. ✅ 升级前备份 workspace 是必要的安全措施
2. ✅ npm 升级过程约 2-3 分钟，需要耐心等待

1. 建议配置 systemd journal 持久化，便于故障排查
2. 建议在升级脚本中加入自动 warm-up 等待
