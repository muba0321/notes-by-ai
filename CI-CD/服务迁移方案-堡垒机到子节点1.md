## 迁移背景

### 原始架构

```
子节点 1 (38.246.245.39, 海外)
├── Jenkins (8080)
├── Prometheus (9090)
├── Grafana (3000)
└── Nginx (代理本地服务)
```

### 迁移目标

将监控/CI/CD 服务从海外子节点 1 迁移到国内堡垒机，子节点 1 转为纯流量转发节点：

```
子节点 1 (38.246.245.39, 海外)
└── Nginx → SSH 隧道 → 堡垒机 (222.211.80.222, 国内)
                        ├── Jenkins
                        ├── Prometheus
                        ├── Grafana
                        └── Node Exporter
```

### 迁移原因

1. 子节点 1 定位为海外流量转发节点
2. 堡垒机资源空闲（内存可用 3.2G，磁盘可用 26G）
3. 国内访问堡垒机速度更快
4. 集中化管理监控/CI/CD 服务

## 迁移方案

### 阶段划分

| 阶段 | 内容 | 预计时间 |
|------|------|----------|
| 一 | 堡垒机环境准备（Docker 安装） | 10 分钟 |
| 二 | 子节点 1 数据备份 | 15 分钟 |
| 三 | 数据传输到堡垒机 | 10 分钟 |
| 四 | 堡垒机服务部署 | 15 分钟 |
| 五 | 子节点 1 SSH 隧道配置 | 10 分钟 |
| 六 | 子节点 1 Nginx 配置更新 | 5 分钟 |
| 七 | 验证与清理 | 10 分钟 |
| **总计** | | **约 75 分钟** |

### 关键技术点

1. **Docker 镜像迁移** - 使用 `docker save/load` 避免网络拉取失败
2. **数据完整性** - 备份 Prometheus WAL 和 Jenkins home 目录
3. **跨境流量加密** - SSH 隧道避免防火墙内容篡改
4. **服务持久化** - systemd 管理 SSH 隧道，断线自动重连

## 详细步骤

### 阶段一：堡垒机环境准备

#### 1.1 安装 Docker

```bash
# 堡垒机执行
ssh root@222.211.80.222

# 使用阿里云镜像源安装 Docker
curl -fsSL https://get.docker.com | sh

# 验证安装
docker --version
systemctl enable docker && systemctl start docker
```

#### 1.2 创建目录结构

```bash
# 堡垒机执行
mkdir -p /data/monitoring/{prometheus,grafana,node-exporter,jenkins}
```

#### 1.3 创建 Docker Compose 配置

```yaml
# /data/monitoring/docker-compose.yml
services:
  prometheus:
    image: prom/prometheus:latest
    container_name: prometheus
    restart: unless-stopped
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus/data:/prometheus
      - ./prometheus/prometheus.yml:/etc/prometheus/prometheus.yml
    command:
      - '--config.file=/etc/prometheus/prometheus.yml'
      - '--storage.tsdb.path=/prometheus'
    networks:
      - monitoring

  grafana:
    image: grafana/grafana:latest
    container_name: grafana
    restart: unless-stopped
    ports:
      - "3000:3000"
    volumes:
      - ./grafana/data:/var/lib/grafana
    environment:
      - GF_SECURITY_ADMIN_USER=admin
      - GF_SECURITY_ADMIN_PASSWORD=admin123
    depends_on:
      - prometheus
    networks:
      - monitoring

  node-exporter:
    image: prom/node-exporter:latest
    container_name: node-exporter
    restart: unless-stopped
    ports:
      - "9100:9100"
    volumes:
      - /proc:/host/proc:ro
      - /sys:/host/sys:ro
      - /:/rootfs:ro
    command:
      - '--path.procfs=/host/proc'
      - '--path.sysfs=/host/sys'
    networks:
      - monitoring

  jenkins:
    image: jenkins/jenkins:lts-jdk17
    container_name: jenkins
    restart: unless-stopped
    ports:
      - "8080:8080"
      - "50000:50000"
    volumes:
      - ./jenkins/home:/var/jenkins_home
    environment:
      - JAVA_OPTS=-Djava.awt.headless=true
    networks:
      - monitoring

networks:
  monitoring:
    driver: bridge
```

#### 1.4 创建 Prometheus 配置

```yaml
# /data/monitoring/prometheus/prometheus.yml
global:
  scrape_interval: 15s
  evaluation_interval: 15s

scrape_configs:
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

  - job_name: 'node-exporter'
    static_configs:
      - targets: ['localhost:9100']
```

### 阶段四：堡垒机服务部署

#### 4.1 解压备份数据

```bash
# 堡垒机执行
cd /data/monitoring

tar xzf prometheus-backup.tar.gz
tar xzf jenkins-backup.tar.gz
tar xzf grafana-backup.tar.gz

# 创建数据目录并移动数据
mkdir -p prometheus/data jenkins/home grafana/data
mv prometheus-backup/* prometheus/data/
mv jenkins-backup/* jenkins/home/
mv grafana-backup/* grafana/data/
```

#### 4.2 修复权限

```bash
# 堡垒机执行
chown -R 472:472 prometheus/data  # Prometheus 用户
chown -R 472:472 grafana/data     # Grafana 用户
chown -R 1000:1000 jenkins/home   # Jenkins 用户

# 删除 Prometheus 锁文件
rm -f prometheus/data/queries.active
```

#### 4.3 导出并传输 Docker 镜像

```bash
# 子节点 1 执行（导出镜像）
cd /backup
docker save prom/prometheus:latest -o prometheus-image.tar
docker save grafana/grafana:latest -o grafana-image.tar
docker save prom/node-exporter:latest -o node-exporter-image.tar
docker save jenkins/jenkins:lts-jdk17 -o jenkins-image.tar

# 传输到堡垒机
scp *.tar root@222.211.80.222:/data/monitoring/

# 堡垒机执行（导入镜像）
cd /data/monitoring
docker load -i prometheus-image.tar
docker load -i grafana-image.tar
docker load -i node-exporter-image.tar
docker load -i jenkins-image.tar

# 验证镜像
docker images | grep -E 'jenkins|prometheus|grafana|node-exporter'
```

#### 4.4 启动容器

```bash
# 堡垒机执行
cd /data/monitoring
docker compose up -d

# 验证容器状态
docker ps --format 'table {{.Names}}\t{{.Status}}\t{{.Ports}}'

# 验证服务
curl -I http://localhost:8080/login   # Jenkins
curl -I http://localhost:9090         # Prometheus
curl -I http://localhost:3000         # Grafana
```

### 阶段五：子节点 1 SSH 隧道配置

#### 5.1 安装依赖

```bash
# 堡垒机执行
apt-get install -y sshpass

# 子节点 1 执行
apt-get install -y autossh sshpass
```

#### 5.2 创建 systemd 服务

```bash
# 子节点 1 执行
cat > /etc/systemd/system/ssh-tunnel.service << 'EOF'
[Unit]
Description=SSH Tunnel to Bastion
After=network.target

[Service]
Type=simple
User=root
ExecStart=/usr/bin/sshpass -p 'Huanxin0321' ssh -o StrictHostKeyChecking=no -o ServerAliveInterval=30 -o ServerAliveCountMax=3 -N -L 127.0.0.1:18080:localhost:8080 -L 127.0.0.1:19090:localhost:9090 -L 127.0.0.1:13000:localhost:3000 root@222.211.80.222
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

# 启用并启动服务
systemctl daemon-reload
systemctl enable ssh-tunnel
systemctl restart ssh-tunnel

# 验证服务状态
systemctl status ssh-tunnel --no-pager

# 验证隧道端口
ss -tlnp | grep -E '18080|19090|13000'

# 测试隧道连接
curl -I http://127.0.0.1:18080/login   # Jenkins
curl -I http://127.0.0.1:19090         # Prometheus
curl -I http://127.0.0.1:13000         # Grafana
```

```bash

server {
    listen 80;
```

## 问题与解决方案

### 问题 1：Docker Hub 拉取超时

**现象：**
```
Error response from daemon: failed to resolve reference "docker.io/grafana/grafana:latest": 
dial tcp 199.59.150.49:443: i/o timeout
```

**原因：** 堡垒机在国内，访问 Docker Hub 网络不稳定

**解决方案：**
```bash
# 方案 A：配置 Docker 镜像加速器
cat > /etc/docker/daemon.json << 'EOF'
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://registry.docker-cn.com"
  ]
}
EOF
systemctl daemon-reload && systemctl restart docker

# 方案 B：从子节点 1 导出镜像传输（推荐）
# 子节点 1 执行
docker save grafana/grafana:latest -o grafana-image.tar
scp grafana-image.tar root@堡垒机:/data/monitoring/

# 堡垒机执行
docker load -i grafana-image.tar
```

### 问题 2：Prometheus 启动失败 - 权限错误

**现象：**
```
panic: Unable to create mmap-ed active query log
err="open /prometheus/queries.active: permission denied"
```

**原因：** 备份的数据目录权限与容器用户不匹配

**解决方案：**
```bash
# 堡垒机执行
# Prometheus 容器用户 UID=472
chown -R 472:472 /data/monitoring/prometheus/data

# 删除锁文件
rm -f /data/monitoring/prometheus/data/queries.active

# 重启容器
docker restart prometheus
```

### 问题 4：SSH 隧道端口已被占用

**现象：**
```
bind [127.0.0.1]:18080: Address already in use
channel_setup_fwd_listener_tcpip: cannot listen to port: 18080
```

**原因：** 之前的 SSH 隧道进程未完全清理

**解决方案：**
```bash
# 子节点 1 执行
# 查找并杀死占用端口的进程
fuser -k 18080/tcp 19090/tcp 13000/tcp

# 或者杀死所有相关 SSH 进程
pkill -f 'ssh.*222.211.80.222'

# 等待端口释放
sleep 3

# 重新启动隧道
systemctl restart ssh-tunnel
```

---

### 问题 5：跨境 HTTP 流量被防火墙篡改

**现象：** 从子节点 1 直接访问堡垒机 HTTP 端口时返回中文拦截页面：
```
温馨提示：该网站暂无法访问
(code:Iawy0Hjxzp10tz)
原因一：根据《非经营性互联网信息服务备案管理办法》，您尚未进行备案或接入
```

**原因：** 海外→国内的 HTTP 明文流量被防火墙检测并篡改

**解决方案：** 使用 SSH 加密隧道
```bash
# SSH 隧道加密传输，防火墙无法检测内容
ssh -L 127.0.0.1:18080:localhost:8080 root@222.211.80.222

# Nginx 代理到本地隧道端口，不直接访问堡垒机
proxy_pass http://127.0.0.1:18080;
```

**架构优势：**
- 用户 → 子节点 1（海外 IP，无拦截）
- 子节点 1 → 堡垒机（SSH 加密，不可检测）
- 全链路不暴露堡垒机 IP

### 问题 6：Jenkins 权限警告

**现象：**
```
Can not write to /var/jenkins_home/copy_reference_file.log. Wrong volume permissions?
INSTALL WARNING: User: missing rw permissions on JENKINS_HOME
```

**原因：** Jenkins 容器用户 UID=1000，数据目录权限不匹配

**解决方案：**
```bash
# 堡垒机执行
chown -R 1000:1000 /data/monitoring/jenkins/home

# 重启容器
docker restart jenkins
```
