## 🟠 中等问题

### OpenSSH 版本兼容性问题

**时间：** 2026-03-18  
**影响范围：** Ansible 连接 K8s 集群机器

#### 问题描述

从子节点 1 (38.246.245.39) 通过 Ansible 连接 4 台 K8s 机器时，SSH 连接被主动关闭：

```
master1 | UNREACHABLE! => {
    "msg": "Failed to connect to the host via ssh: Connection closed by 124.132.136.17 port 9005"
}
```

#### 环境信息

| 组件 | 版本 |
|------|------|
| 子节点 1 SSH 客户端 | OpenSSH 8.9p1 (2022 年) |
| 远程机器 SSH 服务器 | OpenSSH 7.4 (2016 年) |
| 连接方式 | 公网 IP + 端口映射 |

#### 排查过程

1. ✅ 端口连通性正常（telnet 可连接）
2. ✅ 能看到 SSH 横幅（SSH-2.0-OpenSSH_7.4）
3. ❌ SSH 握手阶段被服务器主动关闭

**SSH 调试日志：**
```
debug1: Remote protocol version 2.0, remote software version OpenSSH_7.4
debug1: kex: algorithm: curve25519-sha256
debug1: expecting SSH2_MSG_KEX_ECDH_REPLY
Connection closed by 124.132.136.17 port 9005
```

#### 原因分析

- 远程服务器使用 OpenSSH 7.4（2016 年老版本）
- 控制端使用 OpenSSH 8.9p1（2022 年新版本）
- 密钥交换阶段（KEX）兼容性问题
- 笔记本（老 SSH 客户端）可以连接，子节点 1（新 SSH 客户端）无法连接

#### 尝试的解决方案

| 方案 | 命令 | 结果 |
|------|------|------|
| 添加兼容算法 | `-o KexAlgorithms=+diffie-hellman-group-exchange-sha256` | ❌ 失败 |
| 添加更多算法 | `-o KexAlgorithms=+diffie-hellman-group14-sha1,diffie-hellman-group-exchange-sha1` | ❌ 失败 |
| 放宽密钥算法 | `-o HostKeyAlgorithms=+ssh-rsa,ssh-dss -o PubkeyAcceptedAlgorithms=+ssh-rsa,ssh-dss` | ❌ 失败 |

#### 待尝试方案

1. **升级远程 SSH 服务器**（推荐）
   ```bash
   # 在远程机器上执行
   yum update openssh-server  # CentOS 7
   ```

2. **使用内网 IP 直连**
   - 如果子节点 1 和 K8s 集群在同一内网
   - 避免经过端口映射和防火墙

3. **SSH 中转**
   - 通过一台中间机器转发连接

#### 当前状态

⏸️ 暂停，等待确认：
- 笔记本 SSH 版本（`ssh -V`）
- 笔记本连接成功的详细命令
- 远程机器内网 IP 信息

#### 教训总结

1. **SSH 版本差异可能导致连接失败** - 新老版本加密算法不兼容
2. **端口通≠SSH 通** - telnet 能连不代表 SSH 握手成功
3. **调试日志很重要** - `ssh -v` 能看到具体失败阶段
4. **记录环境信息** - SSH 版本、连接方式、网络拓扑

## 🟡 轻微问题

### 笔记梳理操作确认流程

**时间：** 2026-03-23  
**影响范围：** 笔记路径 (`/data/openclaw-dist/`) 相关操作

#### 规则说明

当用户要求梳理笔记并记录时，**必须先确认以下信息**，再执行写入操作：

1. **文件主要内容** — 要记录的核心信息
2. **存放路径** — 具体放在哪个目录/文件
3. **文件名** — 新建文件还是追加到现有文件

#### 确认话术示例

> "我计划将 [内容摘要] 记录到 `/data/openclaw-dist/[路径]/[文件名]`，主要内容包含 [要点列表]。确认后再执行？"

#### 原因

- 笔记仓库是 Git 管理的正式文档
- 避免写错位置或覆盖重要内容
- 确保分类符合现有目录规范

#### 教训总结

1. **写前确认** — 涉及外部存储的操作先确认再执行
2. **路径规范** — 遵循现有目录结构分类
3. **用户知情** — 让用户知道文件会写在哪里
