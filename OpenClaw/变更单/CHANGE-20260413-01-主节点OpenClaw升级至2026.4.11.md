## 三、升级步骤

### Step 1: 备份
```bash
# 1.1 备份配置文件
cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.backup.$(date +%Y%m%d_%H%M%S)

# 1.2 备份 workspace 关键文件
cp ~/.openclaw/workspace-consult/MEMORY.md ~/.openclaw/workspace-consult/MEMORY.md.backup.$(date +%Y%m%d_%H%M%S)

# 1.3 记录当前版本
openclaw --version > /tmp/openclaw-version-before.txt 2>&1
```

### Step 2: 执行升级（Gateway 不需要停）
```bash
npm install -g openclaw@latest
```
> ⚠️ 注意：不能先停 Gateway，因为升级命令需要当前 Agent 执行，停了自己就挂了。

### Step 3: 验证升级
```bash
# 3.1 检查版本
openclaw --version

# 3.2 检查配置兼容性
openclaw status

# 3.3 检查插件状态
npm list -g openclaw 2>/dev/null
```

### Step 4: 验证升级
```bash
# 4.1 检查版本
openclaw --version

# 4.2 检查配置兼容性
openclaw status

# 4.3 检查插件状态
npm list -g openclaw 2>/dev/null
```

### Step 5: 启动 Gateway
```bash
openclaw gateway start
# 验证
openclaw gateway status
```

### Step 6: 功能验证
```bash
# 6.1 Gateway 端口监听
ss -tlnp | grep 18789

# 6.2 systemd 服务状态
systemctl --user status openclaw-gateway.service

# 6.3 通道连接状态
openclaw status
```

## 四、回滚方案

如果升级后出现严重问题（Gateway 无法启动、配置不兼容等）：

```bash
# 1. 停止新 Gateway
openclaw gateway stop

# 2. 恢复到旧版本
npm install -g openclaw@2026.3.24

# 3. 恢复配置（如被修改）
cp ~/.openclaw/openclaw.json.backup.<timestamp> ~/.openclaw/openclaw.json

# 4. 重启
openclaw gateway start
```
