## 🎯 子代理类型

### Subagent（轻量级）

**特点：**
- 无需预先配置
- 动态创建
- 任务完成后自动清理
- 适合并行执行简单任务

**创建方式：**
```javascript
sessions_spawn({
  task: "检查 master1 状态",
  mode: "run",      // 一次性运行
  runtime: "subagent"
})
```

### ACP（重量级）

**特点：**
- 需要 agentId
- 可持久化会话
- 适合长期运行的专业任务

**创建方式：**
```javascript
sessions_spawn({
  task: "K8s 部署专员",
  agentId: "main",
  mode: "session",  // 持久会话
  runtime: "acp"
})
```

## 🚀 并行任务执行示例

### 示例 1：并行检查 3 台机器

**用户指令：**
```
并行检查以下机器状态：
1. master1 (124.132.136.17:9005)
2. node1 (124.132.136.17:9191)
3. node2 (124.132.136.17:9053)
```

**执行代码：**
```javascript
// 创建 3 个子代理并行执行
const tasks = [
  { name: "master1", host: "124.132.136.17", port: 9005 },
  { name: "node1", host: "124.132.136.17", port: 9191 },
  { name: "node2", host: "124.132.136.17", port: 9053 }
];

const results = await Promise.all(
  tasks.map(task => 
    sessions_spawn({
      task: `检查 ${task.name} 状态：
        1. SSH 连通性测试
        2. 系统资源检查（CPU/内存/磁盘）
        3. Docker 状态
        4. 网络连通性`,
      label: `检查-${task.name}`,
      mode: "run",
      runtime: "subagent"
    })
  )
);

// 汇总结果
return results.map((r, i) => `${tasks[i].name}: ${r.status}`).join('\n');
```

**预期输出：**
```
✅ 创建 3 个子代理并行检查：

🔹 子代理 A: master1 检查
🔹 子代理 B: node1 检查
🔹 子代理 C: node2 检查

执行中... 预计 2-3 分钟
```

## ⚙️ 配置选项

### sessions_spawn 参数

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| task | ✅ | - | 任务描述 |
| agentId | ❌ | - | ACP 专用 agentId |
| mode | ❌ | run | run（一次性）/ session（持久） |
| runtime | ❌ | subagent | subagent / acp |
| label | ❌ | - | 子代理标签 |
| timeoutSeconds | ❌ | 300 | 超时时间 |
| cleanup | ❌ | delete | 完成后清理 |

### 并发控制

```javascript
// 限制最大并发数
const MAX_CONCURRENT = 5;

// 分批执行大量任务
for (let i = 0; i < tasks.length; i += MAX_CONCURRENT) {
  const batch = tasks.slice(i, i + MAX_CONCURRENT);
  await Promise.all(batch.map(task => spawn(task)));
}
```

## 🚨 注意事项

### 资源限制

1. **最大并发数** - 建议不超过 5 个
2. **内存消耗** - 每个子代理约 200-500MB
3. **CPU 使用** - 避免同时运行 CPU 密集型任务

### 任务依赖

```
❌ 错误：任务 B 需要任务 A 的输出
✅ 正确：任务 A 和 B 相互独立
```

### 错误处理

```javascript
// 捕获子代理失败
try {
  const result = await sessions_spawn({ task: "..." });
} catch (error) {
  console.error(`子代理失败：${error.message}`);
  // 记录失败，继续其他任务
}
```

## 📝 最佳实践

### 适合子代理的任务

- ✅ 独立无依赖的检查任务
- ✅ 批量部署（多节点）
- ✅ 数据查询/备份
- ✅ 并行测试

### 不适合子代理的任务

- ❌ 需要人工确认的任务
- ❌ 强依赖关系任务
- ❌ 超长运行任务（>30 分钟）
- ❌ 需要共享状态的任务

## 🔍 调试技巧

### 查看子代理状态

```javascript
// 列出所有子代理
const agents = await subagents({ action: "list" });
console.log(`活跃子代理：${agents.active.length}`);

// 查看子代理详情
agents.active.forEach(agent => {
  console.log(`- ${agent.label}: ${agent.status}`);
});
```

### 日志跟踪

```javascript
// 获取子代理日志
const history = await sessions_history({
  sessionKey: agent.sessionKey,
  limit: 50
});

// 打印日志
history.messages.forEach(msg => {
  console.log(`[${msg.role}] ${msg.content}`);
});
```
