## 📊 概念总览

| 概念 | 英文名称 | 用途 | 运行位置 |
|------|----------|------|----------|
| **子代理** | Subagent | 并行执行临时任务 | 主 Gateway 内 |
| **子 Agent** | Agent | ACP 专用 Agent | 主 Gateway 内 |
| **子节点 OpenClaw** | Child Gateway | 分布式 Gateway 实例 | 远程服务器 |
| **Node（页面）** | Node | 配对设备（手机/电脑） | 用户设备 |

## 1️⃣ 子代理 (Subagent)

### 定义

**临时创建的任务执行者**，用于并行执行多个独立任务。

### 特点

| 特性 | 说明 |
|------|------|
| **生命周期** | 任务完成后自动销毁 |
| **创建方式** | `sessions_spawn({ runtime: "subagent" })` |
| **用途** | 并行执行简单任务 |
| **资源** | 轻量级，约 200-500MB 内存 |
| **配置** | 无需预先配置 |

### 示例

```javascript
// 创建子代理并行检查 3 台机器
const results = await Promise.all([
  sessions_spawn({ 
    task: "检查 master1 状态",
    runtime: "subagent",
    mode: "run"
  }),
  sessions_spawn({ 
    task: "检查 node1 状态",
    runtime: "subagent",
    mode: "run"
  }),
  sessions_spawn({ 
    task: "检查 node2 状态",
    runtime: "subagent",
    mode: "run"
  })
]);
```

### 查看命令

```bash
# 列出所有子代理
subagents action=list
```

## 2️⃣ 子 Agent (ACP Agent)

### 定义

**ACP 框架的专用 Agent**，用于复杂任务或持久会话。

### 特点

| 特性 | 说明 |
|------|------|
| **生命周期** | 可持久化（session 模式） |
| **创建方式** | `sessions_spawn({ runtime: "acp", agentId: "main" })` |
| **用途** | 复杂任务、专业领域 Agent |
| **资源** | 重量级，完整 Agent 功能 |
| **配置** | 需要 agentId |

### 示例

```javascript
// 创建 ACP 会话（K8s 部署专员）
sessions_spawn({
  task: "你是一名 K8s 部署专家，负责部署和维护 K8s 集群",
  agentId: "main",
  runtime: "acp",
  mode: "session"  // 持久会话
});
```

### 查看可用 Agent

```bash
# 列出可用的 ACP Agents
agents_list
```

## 3️⃣ 子节点 OpenClaw (Child Gateway)

### 定义

**分布式部署的 OpenClaw Gateway 实例**，每个子节点是独立的 OpenClaw 服务。

### 特点

| 特性 | 说明 |
|------|------|
| **生命周期** | 长期运行（systemd 服务） |
| **部署方式** | `openclaw gateway start` |
| **用途** | 分布式部署、就近访问 |
| **资源** | 独立服务器资源 |
| **配置** | 需要独立配置文件 |

## 4️⃣ Node（页面/设备）

### 定义

**配对的物理设备**（手机、电脑），通过 OpenClaw Node App 连接。

| **连接方式** | QR 码配对 / 手动连接 |

## 🔍 常见误区

### 误区 1：子代理 = 子节点

```
❌ 错误：子代理就是子节点
✅ 正确：子代理是临时任务执行者，子节点是独立 Gateway
```

### 误区 2：Node = 子节点

```
❌ 错误：Node 就是子节点
✅ 正确：Node 是用户设备（手机），子节点是服务器
```

### 误区 3：子 Agent = 子代理

```
❌ 错误：子 Agent 和子代理是一样的
✅ 正确：子代理是轻量级临时任务，子 Agent 是 ACP 框架
```

## 📝 使用场景

### 场景 1：并行检查 10 台机器

```
使用：子代理 (Subagent)
创建：10 个子代理实例
结果：并行执行，汇总返回
```

### 场景 2：部署 K8s 专家 Agent

```
使用：ACP Agent
创建：持久会话，专门负责 K8s

```

### 场景 3：国内用户就近访问

```
使用：子节点 Gateway（堡垒机）
部署：在国内服务器部署 Gateway
结果：国内用户访问更快
```

### 场景 4：手机随时随地访问

配对：扫描二维码配对手机
