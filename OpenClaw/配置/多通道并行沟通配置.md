## 📖 并行沟通原理

OpenClaw 支持多通道并行沟通，每个通道独立 session，但共享同一个 Agent 实例。

## 🎯 并行沟通方案

### 方案架构

```
┌─────────────────────────────────────────────────────────┐
│                    OpenClaw Gateway                      │
│                                                          │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  │
│  │  Webchat     │  │   钉钉       │  │  其他通道    │  │
│  │  Session A   │  │  Session B   │  │  Session C   │  │
│  └──────┬───────┘  └──────┬───────┘  └──────┬───────┘  │
│         │                 │                 │           │
│         └─────────────────┼─────────────────┘           │
│                           │                             │
│                  ┌────────▼────────┐                    │
│                  │   Agent Core    │                    │
│                  │  (共享上下文)    │                    │
│                  └─────────────────┘                    │
└─────────────────────────────────────────────────────────┘
```

### 工作流程

1. **独立 Session** - 每个通道有独立的 session ID
2. **共享记忆** - 所有通道共享 MEMORY.md 和记忆文件
3. **独立对话** - 每个通道的对话历史独立
4. **并行处理** - 可同时接收多个通道的指令

## ⚙️ 配置说明

### 通道优先级

OpenClaw 默认按消息到达顺序处理，无优先级区分。

### Session 管理

每个通道独立 session：
- Webchat: session_webchat_xxx
- 钉钉：session_dingtalk_xxx

### 记忆共享

所有通道共享：
- MEMORY.md（长期记忆）
- memory/*.md（项目记忆）
- 记忆文件（projects.md, lessons.md 等）

### 对话隔离

每个通道独立：
- 对话历史
- 上下文窗口
- 临时变量

## 🚀 使用建议

### 最佳实践

1. **重要操作** - 使用 Webchat（有完整界面）
2. **快速查询** - 使用钉钉（移动端方便）
3. **长时间对话** - 固定使用一个通道
4. **紧急通知** - 钉钉（手机推送）

### 避免问题

1. **不要同时发送冲突指令** - 如一个说部署，一个说取消
2. **注意上下文切换** - 不同通道上下文不共享
3. **敏感信息** - 避免在钉钉发送密码等敏感信息

## 🔍 故障排查

### 问题 1：钉钉收不到消息

```bash
# 检查钉钉插件状态
systemctl status openclaw-gateway

# 查看日志
tail -f /tmp/openclaw/openclaw-*.log | grep dingtalk
```

### 问题 2：Webchat 无法访问

```bash
# 检查 Gateway 端口
ss -tlnp | grep 18789

# 检查防火墙
ufw status | grep 18789
```

### 问题 3：消息处理慢

```bash
# 检查系统负载
uptime

# 检查 Agent 状态
docker ps | grep openclaw
```
