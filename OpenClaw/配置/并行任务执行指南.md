# Webchat 并行任务执行指南

**更新时间：** 2026-03-24  
**状态：** 配置说明

---

## 🎯 问题描述

**当前限制：** Webchat 单个 Session 只能顺序执行任务

```
用户："任务 A" → Agent 执行 A（等待完成）→ 用户："任务 B" → Agent 执行 B
```

**需求：** 同时下达多个任务，并行执行

```
用户："任务 A + 任务 B" → Agent 同时执行 A 和 B → 分别返回结果
```

---

## ✅ 解决方案

### 方案一：子代理并行执行（推荐）⭐

#### 工作原理

```
用户指令
   ↓
主 Agent 接收
   ↓
创建子代理 A ──→ 执行任务 A ──┐
                              ├──→ 汇总结果 → 返回用户
创建子代理 B ──→ 执行任务 B ──┘
```

#### 使用方式

**用户指令格式：**
```
并行执行以下任务：
1. 任务 A（详细描述）
2. 任务 B（详细描述）
3. 任务 C（详细描述）
```

**Agent 响应：**
```
收到！正在创建 3 个子代理并行执行：
- 子代理 A: 负责任务 A
- 子代理 B: 负责任务 B
- 子代理 C: 负责任务 C

预计完成时间：5-10 分钟
完成后我会汇总结果通知你。
```

#### 技术实现

```javascript
// 主 Agent 接收任务
const tasks = parseTasks(userMessage);

// 创建子代理并行执行
const subagents = tasks.map(task => 
  sessions_spawn({
    task: task.description,
    mode: "run",
    runtime: "subagent"
  })
);

// 等待所有子代理完成
const results = await Promise.all(subagents);

// 汇总结果返回用户
return summarize(results);
```

### 方案二：后台任务 + 通知

#### 工作原理

```
用户指令
   ↓
主 Agent 接收
   ↓
任务 A → 后台执行（yieldMs）──┐
                            ├──→ 完成后通知用户
任务 B → 后台执行（yieldMs）──┘
```

#### 使用方式

**用户指令：**
```
后台执行以下任务：
1. 部署 K8s（这个需要时间）
2. 检查集群状态
```

**Agent 响应：**
```
任务已在后台执行：

完成后我会通过钉钉通知你。
```

### 方案三：多 Session 分工

#### 工作原理

```
用户
├── Session 1 → 任务 A
├── Session 2 → 任务 B
└── Session 3 → 任务 C
```

#### 使用方式

**打开多个浏览器窗口：**
- 窗口 1 (Chrome): "部署 K8s"
- 窗口 2 (Firefox): "检查集群状态"
- 窗口 3 (Edge): "配置 MHA"

**结果：** 三个任务并行执行

## ⚙️ 配置选项

### 子代理配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| maxConcurrent | 5 | 最大并发子代理数 |
| timeout | 300s | 子代理超时时间 |
| streamTo | parent | 结果返回方式 |
| cleanup | delete | 完成后清理 |

### 任务优先级

```javascript
// 高优先级任务优先执行
const PRIORITY = {
  HIGH: 1,    // 紧急任务
  NORMAL: 2,  // 普通任务

};
```

## 🚨 注意事项

### 资源限制

1. **最大并发数** - 默认 5 个子代理
2. **内存限制** - 每个子代理约 500MB
3. **CPU 限制** - 避免同时运行 CPU 密集型任务

### 任务依赖

```
❌ 错误：任务 B 依赖任务 A 的结果
✅ 正确：任务 A 和 B 独立无依赖
```

### 结果汇总

- 子代理完成后自动汇总
- 失败任务单独标记
- 支持部分成功

## 📞 使用建议

### 适合并行的任务

- ✅ 独立无依赖的任务
- ✅ 批量检查/查询
- ✅ 多节点部署
- ✅ 数据备份

### 不适合并行的任务

- ❌ 有依赖关系的任务
- ❌ 需要人工确认的任务
- ❌ 资源密集型任务（同时运行多个）
